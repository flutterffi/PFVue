# PFVue Folder Guide

This file explains the role of each top-level folder.

## Core Learning Folders

### `foundations/`

Use this folder for:

- HTML
- CSS
- JavaScript basics
- DOM events
- localStorage

Practice style:

- small runnable files
- syntax-first
- beginner-friendly drills

### `intermediate/`

Use this folder for:

- ES6+
- async and fetch
- modules
- browser mini apps

Practice style:

- slightly larger exercises
- multi-file JavaScript
- browser workflow thinking

### `vue3_core/`

Use this folder for:

- reactivity
- computed and watch
- props and emits
- slots
- form binding
- lifecycle

Practice style:

- concept-focused Vue pages
- bridge from JavaScript to Vue

### `advanced/`

Use this folder for:

- composables
- router design
- Pinia thinking
- API layer thinking
- TypeScript thinking
- architecture and testing mindset

Practice style:

- project-oriented concept training
- engineering awareness before full app implementation

## Project Folders

### `projects/`

Use this folder when you want:

- page-level practice
- mini project flows
- more realistic UI and state coordination

Current project order:

1. `01_component_playground`
2. `02_admin_dashboard`
3. `03_real_world_app`

## Formal Application Folder

### `app/`

Use this folder for:

- real Vite + Vue 3 structure
- Router
- Pinia
- composables
- mock API
- separated `views`, `components`, `stores`, and `api`

Practice style:

- closest to a real project
- best place to practice code organization and iteration

## Support Folders

### `docs/`

Use this folder for:

- repository navigation
- official learning links
- learning routes

### `modules/`

Use this folder for:

- toolchain and setup study
- package and config understanding

### `assets/`

Use this folder for:

- shared local assets later if needed
